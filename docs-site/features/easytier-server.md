# EasyTier 服务端

NetPanel 支持运行 EasyTier standalone 服务端，作为虚拟网络的中转节点，供 EasyTier 客户端连接使用。

## 功能概述

- 运行 EasyTier standalone 服务端进程
- 作为网络中转节点，帮助 NAT 后的设备建立连接
- 支持多个虚拟网络同时运行

---

## 配置说明

进入 **EasyTier 服务端** 页面，点击 **新建** 按钮：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 名称 | 字符串 | ✅ | — | 服务端实例名称 |
| 启用 | 布尔 | ✅ | `true` | 是否启用 |
| 监听地址 | 字符串 | ❌ | `0.0.0.0` | 监听 IP |
| 监听端口 | 整数 | ✅ | `11010` | 客户端连接端口 |
| 监听协议 | 枚举 | ❌ | `tcp` | 监听协议：`tcp`、`udp`、`ws`、`wss` |

---

## 配置示例

| 字段 | 值 |
|------|-----|
| 名称 | 中转服务器 |
| 监听端口 | `11010` |
| 监听协议 | `tcp` |

配置完成后，在 EasyTier 客户端的服务器地址中填写：`tcp://本机公网IP:11010`

---

## 防火墙配置

```bash
# 放行 EasyTier 监听端口
ufw allow 11010/tcp
ufw allow 11010/udp
```

---

## 注意事项

::: tip 服务端作用
EasyTier 服务端仅作为中转节点，帮助 NAT 后的设备建立连接。一旦 P2P 直连建立成功，流量将不再经过服务端。
:::

::: info 公网 IP 要求
EasyTier 服务端需要部署在有公网 IP 的服务器上（如云服务器、VPS），才能被其他设备访问。
:::
