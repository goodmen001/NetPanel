# 网络防护 (WAF)

网络防护功能集成了 Coraza WAF（Web Application Firewall），可对经过 NetPanel 的 HTTP 流量进行规则过滤和拦截，防御常见的 Web 攻击。

## 功能概述

- 基于 Coraza WAF 引擎，兼容 OWASP ModSecurity 规则集
- 支持自定义规则
- 可配置拦截模式（检测/拦截）
- 实时查看拦截日志

---

## 配置说明

进入 **网络防护** 页面，点击 **新建** 按钮：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 名称 | 字符串 | ✅ | — | WAF 规则组名称 |
| 启用 | 布尔 | ✅ | `true` | 是否启用 |
| 绑定站点 | 选择 | ✅ | — | 关联的 Caddy 站点 |
| 运行模式 | 枚举 | ✅ | `检测` | `检测`（仅记录）或 `拦截`（记录并拦截） |
| 规则集 | 多选 | ❌ | — | 启用的规则集 |
| 自定义规则 | 文本 | ❌ | — | 自定义 SecRule 规则 |

### 运行模式

| 模式 | 说明 |
|------|------|
| 检测模式 | 仅记录匹配的请求，不拦截，适合初期调试 |
| 拦截模式 | 记录并拦截匹配的请求，返回 403 响应 |

### 内置规则集

| 规则集 | 说明 |
|--------|------|
| OWASP CRS | OWASP 核心规则集，防御 SQL 注入、XSS 等常见攻击 |
| IP 黑名单 | 拦截已知恶意 IP |
| 扫描器检测 | 检测并拦截常见漏洞扫描工具 |

## 配置示例

### 基础 WAF 配置

为 Caddy 站点启用基础 WAF 防护：

| 字段 | 值 |
|------|-----|
| 名称 | 基础防护 |
| 绑定站点 | 选择已配置的 Caddy 站点 |
| 运行模式 | 检测（初期建议先用检测模式） |
| 规则集 | OWASP CRS |

### 自定义规则示例

阻止特定 User-Agent：

```
SecRule REQUEST_HEADERS:User-Agent "@contains sqlmap" \
    "id:1001,phase:1,deny,status:403,msg:'SQLMap detected'"
```

阻止访问特定路径：

```
SecRule REQUEST_URI "@beginsWith /admin" \
    "id:1002,phase:1,deny,status:403,msg:'Admin path blocked'"
```

## 查看拦截日志

在 WAF 规则列表中，点击 **日志** 按钮可查看：
- 触发时间
- 来源 IP
- 请求路径
- 匹配的规则 ID
- 处理结果（检测/拦截）

---

## 注意事项

::: warning 误拦截风险
OWASP CRS 规则集可能会误拦截正常请求（误报）。建议先在**检测模式**下运行一段时间，观察日志，确认无误报后再切换到**拦截模式**。
:::

::: tip 规则调优
如果发现某条规则导致误报，可以在自定义规则中添加例外：

```
SecRuleRemoveById 942100
```
:::
