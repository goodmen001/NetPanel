# 域名解析

域名解析功能允许你直接在 NetPanel 面板中管理 DNS 解析记录，无需登录各服务商控制台，支持增删改查操作。

## 功能概述

- 直接管理域名的 DNS 解析记录
- 支持 A、AAAA、CNAME、MX、TXT、NS 等记录类型
- 操作实时生效
- 支持批量管理

---

## 前置条件

使用域名解析功能前，需要先在 [域名账号](/features/domain-account) 页面配置对应服务商的 API 密钥。

---

## 使用说明

### 查看域名列表

进入 **域名解析** 页面，选择已配置的域名账号，系统会自动列出该账号下的所有域名。

### 管理解析记录

点击域名右侧的 **管理** 按钮，进入该域名的解析记录列表，可以：

- **查看**：列出所有解析记录
- **新建**：添加新的解析记录
- **编辑**：修改现有记录
- **删除**：删除不需要的记录

---

## 解析记录配置

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 主机记录 | 字符串 | ✅ | 子域名前缀，`@` 表示根域名，`*` 表示泛解析 |
| 记录类型 | 枚举 | ✅ | A、AAAA、CNAME、MX、TXT、NS 等 |
| 记录值 | 字符串 | ✅ | 解析目标值 |
| TTL | 整数 | ❌ | 缓存时间（秒），默认 600 |
| 优先级 | 整数 | ❌ | MX 记录的优先级 |

### 记录类型说明

| 类型 | 说明 | 示例值 |
|------|------|--------|
| `A` | IPv4 地址 | `1.2.3.4` |
| `AAAA` | IPv6 地址 | `2001:db8::1` |
| `CNAME` | 别名，指向另一个域名 | `target.example.com` |
| `MX` | 邮件服务器 | `mail.example.com` |
| `TXT` | 文本记录，常用于验证 | `v=spf1 include:... ~all` |
| `NS` | 域名服务器 | `ns1.example.com` |

---

## 配置示例

### 添加 A 记录（将子域名指向 IP）

| 字段 | 值 |
|------|-----|
| 主机记录 | `home` |
| 记录类型 | `A` |
| 记录值 | `1.2.3.4` |
| TTL | `600` |

效果：`home.example.com` → `1.2.3.4`

### 添加 CNAME 记录

| 字段 | 值 |
|------|-----|
| 主机记录 | `www` |
| 记录类型 | `CNAME` |
| 记录值 | `example.com` |

效果：`www.example.com` → `example.com`

### 添加 TXT 记录（域名验证）

| 字段 | 值 |
|------|-----|
| 主机记录 | `_acme-challenge` |
| 记录类型 | `TXT` |
| 记录值 | `验证字符串` |

---

## 注意事项

::: warning DNS 传播延迟
DNS 记录修改后，由于全球 DNS 缓存的存在，可能需要几分钟到几小时才能在所有地区生效。TTL 值越小，传播越快，但对 DNS 服务器的查询压力越大。
:::

::: tip 泛解析
使用 `*` 作为主机记录可以创建泛解析，匹配所有未单独配置的子域名。例如 `*.example.com` 可以匹配 `a.example.com`、`b.example.com` 等。
:::
