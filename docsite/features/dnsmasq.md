# 解析服务 (DNSMasq)

解析服务功能内置了 DNS 服务器（基于 DNSMasq），支持自定义 DNS 解析规则和上游 DNS 配置，可用于局域网内的 DNS 服务。

## 功能概述

- 提供局域网 DNS 服务
- 支持自定义域名解析规则（hosts 规则）
- 支持配置上游 DNS 服务器
- 支持 DNS 缓存，加速解析

---

## 配置说明

进入 **解析服务** 页面：

### 基础配置

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 启用 | 布尔 | ✅ | `false` | 是否启用 DNS 服务 |
| 监听地址 | 字符串 | ❌ | `0.0.0.0` | DNS 服务监听 IP |
| 监听端口 | 整数 | ❌ | `53` | DNS 服务监听端口 |
| 缓存大小 | 整数 | ❌ | `150` | DNS 缓存条目数量 |

### 上游 DNS 配置

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 上游 DNS | 字符串 | ❌ | `8.8.8.8` | 上游 DNS 服务器，每行一个 |

常用上游 DNS：
- `8.8.8.8`（Google）
- `1.1.1.1`（Cloudflare）
- `223.5.5.5`（阿里云）
- `119.29.29.29`（腾讯云）

### 自定义解析规则

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Hosts 规则 | 文本 | ❌ | 自定义域名解析，格式同 `/etc/hosts` |
| 额外配置 | 文本 | ❌ | 高级 DNSMasq 配置项 |

---

## 配置示例

### 基础配置

```
监听端口: 53
上游 DNS:
  223.5.5.5
  8.8.8.8
```

### 自定义 Hosts 规则

将内网服务映射到域名，方便局域网内访问：

```
# 格式：IP 域名
192.168.1.100  nas.local
192.168.1.1    router.local
10.144.144.1   home-server.local
```

配置后，局域网内的设备将 DNS 指向 NetPanel 所在 IP，即可通过域名访问内网服务。

### 将特定域名解析到本地

```
# 将 test.example.com 解析到本机
127.0.0.1  test.example.com
```

---

## 使用场景

1. **局域网内网服务域名**：为内网服务配置友好的域名，如 `nas.local`、`router.local`
2. **广告过滤**：将广告域名解析到 `0.0.0.0`，实现全局广告过滤
3. **开发环境**：将开发域名解析到本地，方便测试

---

## 注意事项

::: warning 端口 53 权限
在 Linux 上，监听 53 端口需要 root 权限或 `CAP_NET_BIND_SERVICE` 能力。如果权限不足，可以改用 5353 等高位端口，然后在路由器上配置 DNS 转发。
:::

::: tip 路由器 DNS 设置
要让局域网内所有设备使用 NetPanel 的 DNS 服务，可以在路由器的 DHCP 设置中，将 DNS 服务器地址改为 NetPanel 所在设备的 IP。
:::
