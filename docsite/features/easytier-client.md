# EasyTier 客户端

EasyTier 是一款基于 Rust 编写的去中心化虚拟局域网工具，NetPanel 集成了 EasyTier 客户端，支持将多台不同地点的设备组成一个虚拟局域网，实现异地组网。

## 功能概述

- 将不同地点的设备组成虚拟局域网
- 支持配置虚拟 IP 地址
- 支持多种传输协议（TCP、UDP、WebSocket 等）
- 支持通过公共服务器或自建服务器中转
- 提供网络拓扑图、节点列表、隧道状态等可视化信息

## 工作原理

```
设备 A（家里）──→ EasyTier 网络 ←── 设备 B（公司）
                      ↑
               公共/自建服务器（中转）
```

所有加入同一网络（相同网络名称和密码）的设备，都可以通过虚拟 IP 互相访问，就像在同一局域网内一样。

## 配置说明

进入 **EasyTier 客户端** 页面，点击 **新建** 按钮：

### 基础配置

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 名称 | 字符串 | ✅ | — | 实例名称 |
| 启用 | 布尔 | ✅ | `true` | 是否启用 |
| 网络名称 | 字符串 | ✅ | — | 虚拟网络标识，同一网络的设备需填写相同值 |
| 网络密码 | 字符串 | ❌ | — | 网络访问密码，留空表示无密码 |
| 虚拟 IPv4 | 字符串 | ❌ | 自动分配 | 本设备在虚拟网络中的 IP，如 `10.144.144.1` |

### 服务器配置

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 服务器地址 | 字符串 | ❌ | 公共服务器 | 中转服务器地址，格式：`tcp://ip:port` 或 `udp://ip:port` |

支持的服务器地址格式：
- `tcp://1.2.3.4:11010` — TCP 连接
- `udp://1.2.3.4:11010` — UDP 连接
- `ws://1.2.3.4:11010` — WebSocket 连接
- `wss://1.2.3.4:11010` — WebSocket over TLS

### 监听配置

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 监听协议 | 枚举 | ❌ | `tcp` | 本地监听协议 |
| 监听端口 | 整数 | ❌ | `11010` | 本地监听端口（用于其他节点直连） |

### 高级选项

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 禁用 P2P | 布尔 | ❌ | `false` | 禁用 P2P 直连，所有流量走服务器中转 |
| 禁用加密 | 布尔 | ❌ | `false` | 禁用数据加密（不推荐） |
| 启用 IPv6 | 布尔 | ❌ | `false` | 启用 IPv6 虚拟网络 |
| 延迟优先 | 布尔 | ❌ | `false` | 优先选择延迟最低的路径 |

## 配置示例

### 示例：三台设备异地组网

**设备 A（家里 NAS）：**
| 字段 | 值 |
|------|-----|
| 名称 | 家庭 NAS |
| 网络名称 | `my-home-network` |
| 网络密码 | `secret123` |
| 虚拟 IPv4 | `10.144.144.1` |

**设备 B（公司电脑）：**
| 字段 | 值 |
|------|-----|
| 名称 | 公司电脑 |
| 网络名称 | `my-home-network` |
| 网络密码 | `secret123` |
| 虚拟 IPv4 | `10.144.144.2` |

**设备 C（云服务器，作为中转）：**
配置 [EasyTier 服务端](/features/easytier-server)，然后在设备 A 和 B 的服务器地址中填写：`tcp://云服务器IP:11010`

配置完成后，设备 B 可通过 `10.144.144.1` 访问家里 NAS 的所有服务。

## 网络状态查看

NetPanel 提供了多个 EasyTier 状态查看页面：

- **节点列表**：查看所有在线节点及其虚拟 IP、延迟信息
- **网络拓扑**：可视化展示节点间的连接关系
- **隧道状态**：查看各节点间的隧道类型（P2P/中转）和流量统计
- **网络事件**：查看节点加入/离开等事件日志

## 注意事项

::: warning TUN 设备权限
EasyTier 需要创建 TUN 虚拟网卡，在 Linux 上需要 `NET_ADMIN` 权限或 root 权限。Docker 部署时需要添加 `--cap-add NET_ADMIN` 和 `--device /dev/net/tun`。
:::

::: tip 公共服务器
如果没有自建服务器，可以使用 EasyTier 官方提供的公共服务器（不填写服务器地址时默认使用）。公共服务器仅用于中转，数据经过加密，但建议敏感场景自建服务器。
:::

::: info EasyTier 版本
NetPanel 发布包中已内置对应平台的 EasyTier 二进制文件，无需单独下载。
:::
