# 网络唤醒 (WOL)

网络唤醒（Wake-on-LAN）功能可以通过发送 Magic Packet 远程唤醒局域网内处于休眠或关机状态的设备。

## 功能概述

- 发送 Magic Packet 唤醒局域网设备
- 支持保存常用设备，一键唤醒
- 支持指定广播地址和端口

## 工作原理

Magic Packet 是一种特殊的网络广播包，包含目标设备的 MAC 地址。当支持 WOL 的网卡接收到 Magic Packet 后，会触发设备开机。

**前提条件：**
1. 目标设备的网卡支持 Wake-on-LAN
2. 在 BIOS/UEFI 中启用 WOL 功能
3. 发送方与目标设备在同一局域网（或通过广播地址可达）

---

## 配置说明

进入 **网络唤醒** 页面，点击 **新建** 按钮添加设备：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| 名称 | 字符串 | ✅ | — | 设备名称，如"家里电脑" |
| MAC 地址 | 字符串 | ✅ | — | 目标设备的 MAC 地址 |
| 广播地址 | 字符串 | ❌ | `255.255.255.255` | 广播地址，局域网内通常使用默认值 |
| 端口 | 整数 | ❌ | `9` | WOL 端口，通常为 7 或 9 |

### MAC 地址格式

支持以下格式：
- `AA:BB:CC:DD:EE:FF`（冒号分隔）
- `AA-BB-CC-DD-EE-FF`（连字符分隔）
- `AABBCCDDEEFF`（无分隔符）

---

## 使用示例

### 添加设备

| 字段 | 值 |
|------|-----|
| 名称 | 家里台式机 |
| MAC 地址 | `AA:BB:CC:DD:EE:FF` |
| 广播地址 | `192.168.1.255` |
| 端口 | `9` |

### 发送唤醒包

在设备列表中，点击设备右侧的 **唤醒** 按钮，即可立即发送 Magic Packet。

---

## 注意事项

::: tip 获取 MAC 地址
- **Windows**：在命令提示符中运行 `ipconfig /all`，查看"物理地址"
- **Linux**：运行 `ip link show` 或 `cat /sys/class/net/eth0/address`
- **macOS**：在"系统偏好设置 → 网络 → 高级 → 硬件"中查看
:::

::: warning 跨网段唤醒
WOL 默认只在同一广播域内有效。如果需要跨网段唤醒，需要路由器支持 WOL 中继，或使用定向广播地址（如 `192.168.1.255`）。
:::

::: info 唤醒失败排查
如果唤醒失败，请检查：
1. 目标设备 BIOS 中是否启用了 WOL
2. 操作系统是否允许网卡在关机后保持供电
3. 防火墙是否阻止了 UDP 9 端口
:::
